# Telegram Knowledge Base Bot

Бот для технической поддержки с использованием RAG (Retrieval-Augmented Generation).

## Возможности

- 🤖 Интеграция с Telegram
- 📚 Индексация текстовых документов
- 🧠 Выбор между OpenRouter и локальными моделями для эмбеддингов и LLM
- 💾 Сохранение истории запросов пользователей
- 🐳 Поддержка Docker

## Быстрый старт

1. Клонируйте репозиторий:
```bash
git clone <repository-url>
cd telegram-knowledge-bot
```

2. Скопируйте и настройте .env:
```bash
cp .env.example .env
# Отредактируйте .env, добавив ваш TELEGRAM_BOT_TOKEN и OPENROUTER_API_KEY
```

3. Создайте папку для базы знаний:
```bash
mkdir knowledge_base
# Добавьте ваши текстовые файлы (.txt, .md) в эту папку
```

4. Запустите бота:
```bash
# Только бот (использует OpenRouter)
docker-compose up

# С локальной LLM (требуется GPU)
docker-compose --profile with-local-llm up
```

## Конфигурация

### Переменные окружения (.env)

- `TELEGRAM_BOT_TOKEN` - токен вашего Telegram бота
- `OPENROUTER_API_KEY` - API ключ OpenRouter
- `EMBEDDING_METHOD` - метод эмбеддингов: `openrouter` или `local`
- `LLM_METHOD` - метод LLM: `openrouter` или `local`

### Локальная LLM

Для использования локальной LLM рекомендуется:
1. **vLLM** (включен в docker-compose) - быстрая инференс-система с поддержкой OpenAI API
2. **Ollama** - альтернативный вариант

## Команды бота

- `/start` - приветственное сообщение
- `/rebuild` - перестроить индекс базы знаний (только для админов)

## Структура проекта

```
.
├── bot/ # Код бота
├── knowledge_base/ # Папка с документами (создается вручную)

├── chroma_db/ # База данных ChromaDB для эмбеддингов
```

## Инструкция по развертыванию

1. **Создайте Telegram бота** через @BotFather и получите токен

2. **Получите API ключ** на [OpenRouter](https://openrouter.ai/)

3. **Подготовьте базу знаний**: создайте папку `knowledge_base` и положите туда текстовые файлы

4. **Настройте .env файл** с вашими данными

5. **Запустите бота**:
```bash
docker-compose up -d
```

6. **Для использования локальной LLM** (требуется GPU с CUDA):
```bash
docker-compose --profile with-local-llm up -d
```

## Особенности реализации

1. **Два способа индексации**:
   - OpenRouter: использует API для эмбеддингов
   - Локальный: использует sentence-transformers

2. **Автоматический выбор стратегии**: бот сам решает, использовать ли эмбеддинги или весь контекст

3. **Сохранение истории**: все диалоги сохраняются в папке `user_queries` с разделением по пользователям

4. **Структурированные ответы**: ответы форматируются с указанием источников

Бот готов к использованию! Для добавления новых документов просто положите их в папку `knowledge_base` и выполните команду `/rebuild` в чате с ботом.
